总结

总的来说event和rpc、api，有些相似。我们来对比下

  1. api 需要定义每一个服务名称对应的方法和参数，event可以通过事件名称获取。比如上边的方法名可以理解为Event1.Get。
  2. api 每一个服务只有一个响应方法，event 可以有多个。
  3. api 可以响应返回数据，event 不可以。所以event 可以用作一些无需要返回数据的业务，如日志提交接口。

其他差异需要你们细细去对比了。
